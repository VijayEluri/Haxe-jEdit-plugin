HaXe plugin

(c) 2010 by Dion Amago

Provides a SideKick structure tree and code completion for HaXe scripts. Also provides utility functions for building and launching e.g. swfs. The HaXe mode is automatically copied into the mode folder in your jEdit settings and the catalog.xml file is edited.

Additional installation steps:

Ctags parsing: ctags support for HaXe is pretty weak. Append this to your .ctags file (or create it if it doesn't exist (~/.ctags on linux and mac).

Credit to Gershon

.

--langdef=haxe
--langmap=haxe:.hx

--regex-haxe=/^[ \t]*import[ \t]+([A-Za-z0-9_.]+)[ \t]*; /\1 /F,import, imports/
--regex-haxe=/^[ \t]*enum[ \t]+([A-Za-z0-9_]+).* /\1 /e,enum, enumerators/
--regex-haxe=/^[ \t]*typedef[ \t]+([A-Za-z0-9_]+)[ \t]+.* /\1 /t,typedef, typedefs/
--regex-haxe=/^[ \t]*((private|public|static|inline)[ \t])*[ \t]+var[ \t]+([A-Za-z0-9_]+)[ \t]*(: *([A-Za-z0-9_]+))? *(=|;)/\3 : \5/v,variable, variables/
--regex-haxe=/^[ \t]*((private|public|static|override) *)*[ \t]*function[ \t]+(([A-Za-z0-9_]+)[ \t]*\(+([A-Za-z0-9_:<> ]*)*\).*)/\3/f,function, functions/
--regex-haxe=/^[ \t]*[(class|interface) ( \t)]+([A-Za-z0-9_]+)+([ \t]?)(extends|implements)[ \t]+([A-Za-z0-9_]+)+[ \t]*/\1 \2 /c,class, classes/

The HaXe compiler needs to be specified in the plugin options. The default points to the mac haxe binary. The HaXe compiler provides the code completion functionality.

Like all "SideKick" plugins, this has no dockable view. Instead, it shares its view with other SideKick plugins through the SideKick Structure Browser. To use SideKick, dock the Structure Browser on the left or the right. 

Global Options -> Docking -> Structure Browser
