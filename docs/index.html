HaXe plugin

(c) 2010 by Dion Amago
Haxe editing mode: Kostas Michalopoulos

Provides code completion and (optionally) a code browser for the HaXe language. Also provides utility functions for building and launching e.g. swfs.

If you have Haxe installed, code completion should work out of the box. The HaXe mode is automatically copied into the mode folder in your jEdit settings and the catalog.xml file is edited.

Additional (optional) installation steps:

The Haxe plugin can use the CtagsSideKick plugin to parse and display HaXe code in navigable browser. Ctags doesn't support HaXe out of the box. Append this to your ~/.ctags file (or create it if it doesn't exist:

--langdef=haxe
--langmap=haxe:.hx
--regex-haxe=/^[ \t]*import[ \t]+([A-Za-z0-9_.]+)[ \t]*; /\1 /F,import, imports/
--regex-haxe=/^[ \t]*enum[ \t]+([A-Za-z0-9_]+).* /\1 /e,enum, enumerators/
--regex-haxe=/^[ \t]*typedef[ \t]+([A-Za-z0-9_]+)[ \t]+.* /\1 /t,typedef, typedefs/
--regex-haxe=/^[ \t]*((private|public|static|inline)[ \t])*[ \t]+var[ \t]+([A-Za-z0-9_]+)[ \t]*(: *([A-Za-z0-9_]+))? *(=|;)/\3 : \5/v,variable, variables/
--regex-haxe=/^[ \t]*((private|public|static|override) *)*[ \t]*function[ \t]+(([A-Za-z0-9_]+)[ \t]*\(+([A-Za-z0-9_:<> ]*)*\).*)/\3/f,function, functions/
--regex-haxe=/^[ \t]*(class|interface)[ \t]+([A-Za-z0-9_]+)[ \t]*($|((extends|implements)?[ \t]+([A-Za-z0-9_]+))?.*)/\2 \5 \6 /c,class, classes/
